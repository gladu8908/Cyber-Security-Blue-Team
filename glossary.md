# SOC Glossary

Core terms used in Security Operations. Expand this file as you encounter new terminology.

---

## A

**Alert**  
A notification generated by a security tool (SIEM, EDR, IDS) indicating a potentially suspicious event that requires analyst attention.

**ATT&CK (MITRE ATT&CK)**  
A knowledge base of adversary tactics, techniques, and procedures (TTPs) based on real-world observations. Used to describe and classify attacker behaviour.

---

## B

**Baseline**  
The normal, expected behaviour of a system or user. Deviations from baseline can indicate suspicious activity.

**Blue Team**  
The defensive side of cybersecurity: detection, monitoring, incident response, and hardening.

---

## C

**CIRT / CSIRT**  
Computer (Security) Incident Response Team — the group responsible for handling security incidents.

**CVE**  
Common Vulnerabilities and Exposures — a standardised identifier for a publicly known security vulnerability (e.g., CVE-2021-44228).

---

## D

**Detection Engineering**  
The discipline of writing, testing, and maintaining detection rules (SIEM rules, Sigma, Yara) to identify threats.

**DFIR**  
Digital Forensics and Incident Response — combining forensic investigation techniques with incident handling.

---

## E

**EDR**  
Endpoint Detection and Response — security software installed on endpoints that records activity and can respond to threats (e.g., CrowdStrike Falcon, SentinelOne).

**Event Log**  
A record of activity on a system. In Windows, key logs include the Security, System, Application, and Sysmon logs.

---

## F

**False Negative**  
A real attack that was NOT detected (missed detection).

**False Positive**  
An alert that fired but no real threat exists — benign activity triggered a detection rule.

---

## I

**IDS / IPS**  
Intrusion Detection System / Intrusion Prevention System — monitors network or host traffic for known attack signatures or anomalies. IDS alerts only; IPS can block.

**IOC (Indicator of Compromise)**  
An artifact that suggests a system may have been compromised: IP address, domain, file hash, registry key, URL.

**IR (Incident Response)**  
The structured process for detecting, containing, eradicating, and recovering from a security incident. Common framework: Prepare → Identify → Contain → Eradicate → Recover → Lessons Learned.

---

## K

**KQL**  
Kusto Query Language — the query language used in Microsoft Sentinel and Azure Log Analytics.

---

## L

**Lateral Movement**  
Attacker behaviour where they move from one compromised system to other systems within the same network to expand access.

**Least Privilege**  
Security principle: users and systems should have only the minimum permissions needed to perform their function.

---

## M

**MITRE ATT&CK**  
See *ATT&CK*.

**MSSP**  
Managed Security Service Provider — a third-party company providing outsourced SOC / security monitoring services.

---

## P

**Persistence**  
A MITRE ATT&CK tactic: techniques adversaries use to maintain their foothold on a compromised system across reboots or credential changes (e.g., scheduled tasks, registry run keys).

**Playbook**  
A documented, step-by-step response procedure for a specific type of alert or incident (e.g., "Ransomware Playbook").

**Privilege Escalation**  
Gaining higher-level permissions than initially obtained — e.g., going from standard user to local administrator.

---

## R

**Red Team**  
The offensive side: simulating real-world attackers to test defences (contrast with Blue Team).

---

## S

**SIEM**  
Security Information and Event Management — a platform that collects, normalises, correlates, and alerts on log data from across the environment (e.g., Splunk, Microsoft Sentinel, IBM QRadar).

**SOC**  
Security Operations Centre — a centralised team monitoring and responding to security events 24/7.

**SPL**  
Search Processing Language — the query language used in Splunk.

**Sysmon**  
System Monitor — a free Windows Sysinternals tool that extends Windows event logging with detailed process, network, file, and registry activity.

---

## T

**Threat Hunting**  
Proactively searching through data to find threats that have evaded automated detections.

**Triage**  
The process of quickly evaluating an alert to determine its priority, validity, and required next action.

**TTP**  
Tactics, Techniques, and Procedures — the behaviour patterns and methods used by threat actors.

**True Positive**  
An alert that fired and correctly identifies real malicious activity.

---

## V

**Vulnerability**  
A weakness in a system, software, or process that could be exploited by a threat actor.
